BOM Location Check vs XY / Schematic

For every assembled location required by the BOM, first count its occurrences in the assigned PCB XY file or schematic. More than one occurrence is reported as a duplicate; one occurrence passes; zero occurrences are reported as not found. Extra XY/schematic locations that are not required by the BOM are ignored.

BOM A vs BOM B + Multi-PCB XY Comparison

Upload BOM A and BOM B, then add and name the XY file for every PCB in the product. The checker classifies BOM locations into Display, Control, MEMS, Power, or any other PCB name you choose, then presents a side-by-side BOM A versus BOM B comparison.

1. Upload Two BOMs
2. Add PCB XY Files
3. Classify BOM Rows
4. Compare & Export
Upload BOM A, BOM B, and at least one named PCB XY file. All processing stays inside your browser.

Step 1 — Upload BOM A and BOM B

Map the same logical fields independently because the two BOM formats may be different.

Step 2 — Configure Product PCBs and XY Files

Add one card for every PCB, rename it as you need, and upload one XY file. The same XY file classifies both BOM A and BOM B using its designators or part numbers. When no unique XY match is found, the PCB column remains blank.

XY formats: `.xlsx`, `.xlsm`, `.csv`, `.txt`, or `.xy`. The required XY field is Designator/Location. Part number, value, package, side, rotation, X and Y are optional.

Step 3 — Classify BOM A and BOM B into PCBs

Locations are assigned from the uploaded XY files. If a designator exists on several PCBs, board/parent text and part-number evidence are used; unresolved rows are shown for manual assignment.

Classification priority: manual selection → unique XY designator → matching XY part/alternative → BOM Board/Parent text → single configured PCB. Rows without locations are assigned by Board/Parent text or manual selection, then compared by part number only.

Step 4 — Compare PCB by PCB

Located items use PCB + location. Approved no-location items use PCB + part number. Alternative lists are included in part matching.

Comparison order: exact main part number across located and no-location rows, approved alternative in either direction, common valid designator, then Add/Delete. Words without letters-and-digits are not treated as locations. A QTY-zero row with only bracketed/not-assembled locations or a blank location passes when absent from the other BOM.
